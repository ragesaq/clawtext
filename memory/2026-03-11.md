## RFC-0001 Model-Persona Behavior Stack - Final Architecture Decisions

After collaborative refinement between Haiku, Opus (Claude), and GPT-5.4, RFC-0001 establishes the Model-Persona Behavior Stack with key decisions:

- **PERSONA layer** is the primary contribution: reusable, model-agnostic behavioral steering as a first-class construct
- **ROLE** concept identified but intentionally deferred to future RFC (ROLE = what job is being performed, PERSONA = how the model behaves while doing that job)
- **ALIAS** remains a convenience label, not a behavioral operating parameter - avoiding unnecessary complexity
- **AGENT** stays at the top as chief operating officer, not redefined in this RFC
- **Novelty claim softened** for public review: "appears to be underdeveloped in current LLM tooling" rather than explicit "novel contribution" language

The stack for multi-agent single-node architecture: PROMPT → ROLE → SESSION (root context) → PERSONA → MODEL → SYSTEM stack. This enables modularity where personas can be traded like skills on clawhub, supporting emergent multi-node multi-agent systems.

Handoff artifacts created: RFC-0001-model-persona-behavior-stack.md, REVISED version, refinement plan, multiple handoff packages between models. Ready for PR and public RFC publication.

## Mental Model: How Embeddings Transform Through Agent Memory

User requested explanation of how data gets stored via embeddings and transforms through an agent's memory stack. Key concepts explored:

- Embeddings start as human-language text transformed into high-dimensional vector space
- Deeper into the memory stack, information becomes increasingly compacted and abstracted
- Transformation moves from explicit language toward symbolic representations (not necessarily hex/glyphs, but semantic vectors)
- Parallel drawn to OpenClaw's own memory system: daily notes (raw) → MEMORY.md (curated) → clusters (indexed/searchable)
- Question about whether representations become "less human-language like" as they go deeper - explored the spectrum from explicit text to dense semantic encoding

This inquiry reflects ongoing work to understand and improve the ClawText memory extraction and clustering system.

## ShaderHook LUT Path Recovery Pattern

Observed working pattern in RageFX ShaderHook for handling malformed LUT paths:

- Malformed paths like `C:\UserssaqAppDataLocalRageFXlutsEZLUT23-65.cube` are automatically recovered to proper format `C:\Users\saq\AppData\Local\RageFX\luts\EZLUT23-65.cube`
- System logs recovery events and continues operation without failure
- Config signature detection prevents unnecessary recompile/hot-swap when unchanged
- Watchdog summarizes operations: createPS, psSet, replacedBinds counts

This demonstrates a robust error-recovery pattern: detect malformed input, normalize it, log the recovery, and continue. Useful pattern for file path handling in other systems.
