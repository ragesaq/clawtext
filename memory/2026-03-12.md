## Discord Agent Tool PR (#43819) — Session Learnings

### GITHUB_TOKEN Auth Issue
- `GITHUB_TOKEN` env var overrides `gh` stored credentials
- If `GITHUB_TOKEN` is set but invalid/expired, `gh` commands fail with 401
- **Fix:** `unset GITHUB_TOKEN` before `gh` commands, or `export GITHUB_TOKEN=""` to blank it
- Stored credentials in `~/.config/gh/hosts.yml` are separate and work fine

### PR Workflow Lessons
1. **Read CONTRIBUTING.md BEFORE creating PR** — it has required template and AI checklist
2. **All required checks:** `pnpm build && pnpm check && pnpm test` before committing
3. **Formatter issues:** `pnpm format` may touch unrelated files (revert with `git checkout`)
4. **PR template:** `.github/pull_request_template.md` — full structured template (Summary, Change Type, Scope, Security Impact, Repro, Evidence, Human Verification, Migration, Failure Recovery, Risks)
5. **AI/Vibe-Coded section:** Must include: title mark, degree of testing, prompts/session logs, understanding confirmation, codex review note, bot review resolution commitment

- `message` tool (in catalog) → cross-provider → send-only via channel plugins

- Issues #458 (Jan 2026) and #6538 (Feb 2026) were open feature requests for this

  3. Lumbot voice command PRD (`2026-03-06T05:30:05Z`)

- Preferred fix order:
  1. Do not fork new forum thread sessions from parent forum channel by default

  3. At minimum, exclude old assistant-authored parent posts from child bootstrap context
