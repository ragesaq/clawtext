# Gold Standard Post Skill — Full Continuity Packet

## Why this handoff exists
This discussion grew out of refining ClawText’s README/product framing. We realized the higher-value reusable outcome is not just “rewrite this README well,” but to create a reusable product-writing skill seeded from a proven gold-standard artifact.

## Current objective
Create a reusable, adaptable skill that can produce strong product-facing writing from a gold-standard seed artifact while preserving structure, excluding irrelevant spillover, and adapting to multiple surfaces.

## Established decisions
- This should become a real skill / repeatable process.
- It should not be rigid or template-prison writing.
- It should start from a proven high-quality artifact and preserve its narrative strengths.
- It should separate real product features from supporting cleanup, repo work, adjacent platform work, and unreleased spillover.
- It should support at least: README, GitHub post, Discord/update post, and showcase framing.
- ClawText’s older gold-standard README is the first concrete seed/reference artifact.

## Open questions
- What exact narrative spine should every gold-standard post start from?
- How do we encode “what belongs vs what does not belong” in a reusable way?
- What output modes should be standard?
- What should the seed format look like so it is modifiable but still strong?
- Should the skill center on README-first generation, or support multiple equal primary modes?
