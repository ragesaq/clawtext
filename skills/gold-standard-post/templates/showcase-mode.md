# Showcase Mode Template

Use this for story-first posts: "Here's what I built and why it matters."

## Instructions
1. Start with your seed artifact
2. Flip the structure: Story → Problem → Solution → Impact
3. More personal tone, less formal
4. Focus on the "why" over the "what"
5. Include the journey (what you tried, what failed, what worked)

## Target Length
1-2 KB

## Formatting
- Conversational Markdown
- Personal anecdotes OK
- Code examples show the "aha moment"
- End on the transformation

## Review Checklist
- [ ] Does it feel like a person talking, not a spec?
- [ ] Is the journey clear (problem → exploration → solution)?
- [ ] Would someone relate to the frustration described?
- [ ] Does it end on the impact/transformation?

## Example Structure
```
I kept running into [problem]. I tried [X], [Y], [Z]. Nothing worked.

Then I realized: [aha moment].

So I built [product]. Here's how it works:

[Simple explanation + one key example]

The result: [transformation — what's different now]

If you've ever [felt this pain], this might help: [link]
```
