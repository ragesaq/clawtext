# Discord Brief Mode Template

Use this for ultra-short, single-post announcements on Discord.

## Instructions
1. Start with your seed artifact
2. Distill to: Hook + Problem + Solution + Link
3. Single paragraph or two max
4. Use emoji + bold sparingly (max 2 emoji)
5. Threadable — should work as a Discord message or thread post

## Target Length
100-200 words (~1,500 characters including markdown)

## Formatting
- No markdown tables or headers (# ##)
- Use **bold** for emphasis
- Emoji only for visual breaks (max 2)
- Links in `<>` to suppress embeds: `<https://example.com>`
- Line breaks for scanability
- Bullet lists OK if very short (3 items max)

## Structure
```
🎙️ **[Product Name]** — [One-line value prop]

[Problem in 1-2 sentences]

[Solution in 1-2 sentences]

[Call to action with link] <https://link>
```

## Review Checklist
- [ ] Can someone scan this in 10 seconds?
- [ ] Is the problem immediately clear?
- [ ] Is the value prop in the first line?
- [ ] Does it fit in one Discord message?
- [ ] Is there a clear link/CTA?

## Examples

**Example 1 — Tight 2-liner:**
```
🌉 **ClawBridge** — Never lose context when threads move

You're deep in an architecture discussion. Thread gets long. New thread starts fresh. Context vanishes. ClawBridge preserves it automatically: one command transfers your decisions, open questions, and next steps.

<https://github.com/ragesaq/clawbridge>
```

**Example 2 — Slightly longer:**
```
**ClawSaver** — Reduce model API costs 20–40%

Users send multiple messages in quick succession. Without batching, that's multiple API calls. With ClawSaver, they batch automatically into one request.

Result: Same quality answers, lower costs. Zero user friction. Install: `npm install clawsaver`

<https://github.com/...>
```

## When to Use This

- Announcing a product in #ai-projects
- Pinning a tool release to a channel
- Sharing in a community Slack/Discord
- Follow-up to a longer thread post
- Product launch in a single message
