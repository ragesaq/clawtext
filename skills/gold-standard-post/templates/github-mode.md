# GitHub Post Mode Template

Use this for GitHub feed announcements, release notes, or shorter product posts.

## Instructions
1. Start with your seed artifact
2. Condense to: Hook → Problem → Solution → Quick Start → Impact
3. Remove deep technical dives (link to full README instead)
4. Focus on the "what changed" and "why it matters"

## Target Length
1-2 KB

## Formatting
- GitHub-flavored Markdown
- Bullet lists instead of long paragraphs
- One key code example
- Link to full documentation

## Review Checklist
- [ ] Can someone scan this in 30 seconds and get it?
- [ ] Is the main value prop obvious?
- [ ] Is there a clear call to action (install, read more, try it)?
- [ ] Does it stand alone without the full README?
